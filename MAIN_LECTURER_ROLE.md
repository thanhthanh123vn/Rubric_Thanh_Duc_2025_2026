# Vai trò Giảng viên Chính - Phân tích so sánh

## 1. Phân loại người dùng trong hệ thống

| Vai trò | Quyền chính | Phạm vi |
|---------|-----------|---------|
| **Admin** | Quản lý toàn bộ hệ thống, người dùng, khóa học | Toàn trường |
| **Main Lecturer** | Tạo CLO, Rubric, Ma trận cho toàn bộ khóa học | Toàn bộ các khóa học của một bộ môn/chuyên đề |
| **Teacher** | Sử dụng rubric, chấm điểm, phản hồi trong một khóa học | Một khóa học cụ thể |
| **Student** | Xem rubric, hoàn thành bài tập, nhận phản hồi | Bài tập cá nhân |

## 2. Trách nhiệm của Giảng viên Chính

### A. Quản lý CLO (Course Learning Outcomes)
```
Quá trình:
1. Xác định chuẩn đầu ra học phần
2. Sắp xếp theo Bloom's Taxonomy
3. Ghi rõ mô tả chi tiết
4. Gửi duyệt
5. Cập nhật dựa trên phản hồi

Kết quả:
- Đảm bảo CLO nhất quán trên tất cả khóa học
- Hỗ trợ các giảng viên khi tạo rubric
- Kiểm soát chất lượng đào tạo
```

### B. Xây dựng Rubric
```
Quá trình:
1. Phân tích CLO → Xác định tiêu chí đánh giá
2. Định nghĩa mức độ thành thạo (proficiency levels)
3. Gán trọng số cho mỗi tiêu chí
4. Tạo mẫu tiêu chuẩn
5. Cung cấp cho giảng viên sử dụng

Đặc điểm:
- Rubric chuẩn hóa, không phụ thuộc vào khóa học cụ thể
- Có thể tùy chỉnh theo loại bài tập (báo cáo, project, thuyết trình)
- Được tái sử dụng bởi nhiều giảng viên
```

### C. Tạo Ma trận Rubric
```
Quá trình:
1. Chọn các CLO cần đánh giá
2. Lựa chọn Rubric phù hợp
3. Ánh xạ CLO → Tiêu chí của Rubric
4. Gán các khóa học sử dụng ma trận này
5. Phê duyệt và công khai

Kết quả:
- Ma trận rõ ràng giữa CLO và tiêu chí chấm điểm
- Đảm bảo Constructive Alignment
- Hỗ trợ giáo dục hướng theo kết quả (OBE)
```

## 3. Luồng công việc

### Scenario: Tạo rubric cho "Dự án Lập trình"

```
┌─────────────────────────────────────────────────────────┐
│ 1. Main Lecturer: Quản lý CLO                          │
│    ✓ CLO1: Thiết kế kiến trúc phần mềm (Vận dụng)      │
│    ✓ CLO2: Viết code chất lượng (Vận dụng)             │
│    ✓ CLO3: Làm việc nhóm hiệu quả (Phân tích)          │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│ 2. Main Lecturer: Tạo Rubric "Dự án Phần mềm"          │
│    Tiêu chí 1: Chất lượng code (40%)                   │
│    Tiêu chí 2: Làm việc nhóm (35%)                     │
│    Tiêu chí 3: Tài liệu & báo cáo (25%)                │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│ 3. Main Lecturer: Tạo Ma trận Rubric                   │
│    CLO1 → Tiêu chí 1 (Chất lượng code)                 │
│    CLO2 → Tiêu chí 1 (Chất lượng code)                 │
│    CLO3 → Tiêu chí 2 (Làm việc nhóm)                   │
│    Áp dụng: Lập trình K65, K66, K67                    │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│ 4. Teachers: Sử dụng Ma trận để chấm điểm              │
│    - Xem CLO, tiêu chí, mức độ                         │
│    - Nhập điểm theo tiêu chí                           │
│    - Hệ thống tự tính điểm tổng                        │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│ 5. Main Lecturer: Báo cáo & Phân tích                  │
│    - Xem hiệu quả đạt CLO trên toàn bộ khóa học        │
│    - Điều chỉnh CLO/Rubric cho năm tới                 │
└─────────────────────────────────────────────────────────┘
```

## 4. So sánh Vai trò

### Main Lecturer vs Teacher

| Khía cạnh | Main Lecturer | Teacher |
|---------|-------------|---------|
| **Quản lý CLO** | ✓ Tạo mới, duyệt | ✓ Xem, áp dụng |
| **Tạo Rubric** | ✓ Tạo mẫu chung | ✓ Tùy chỉnh cho khóa học |
| **Chấm điểm** | ✗ Báo cáo tổng hợp | ✓ Chấm từng bài |
| **Ma trận** | ✓ Tạo ánh xạ chính | ✓ Xem ánh xạ |
| **Phạm vi** | Toàn bộ chuyên đề | Một khóa học |
| **Phê duyệt** | ✓ Quyết định cuối | ✗ Tuân thủ |

## 5. Ưu điểm của mô hình này

✅ **Tập trung chất lượng**: Main Lecturer đảm bảo tiêu chuẩn chung
✅ **Tính nhất quán**: CLO & Rubric chuẩn hóa trên toàn khóa học
✅ **Linh hoạt**: Teacher có thể tùy chỉnh phù hợp lớp học
✅ **Dễ quản lý**: Admin có thể theo dõi từng level
✅ **OBE alignment**: Đảm bảo constructive alignment

## 6. Khuyến cáo triển khai

### Phase 1: Khởi tạo
- Main Lecturer tạo 3-5 CLO cơ bản
- Tạo 2-3 rubric mẫu
- Thử nghiệm với 1-2 khóa học pilot

### Phase 2: Mở rộng
- Mở rộng CLO theo khối ngành
- Tạo thêm rubric cho các loại bài tập khác
- Huấn luyện teachers sử dụng ma trận

### Phase 3: Tối ưu
- Phân tích hiệu quả đạt CLO
- Điều chỉnh rubric dựa trên phản hồi
- Xây dựng thư viện rubric chia sẻ
